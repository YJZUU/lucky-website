# lucky-website
2024 운세 홈페이지 제작

## 🍀 LUCKY 웹사이트
개인 프로젝트 | 2024.09 ~ 12 | https://zuuhome.netlify.app/


하루의 행운을 비는 LUCKY 페이지를 직접 설계·구현하고
SNS를 통해 지인들에게 배포한 개인 프로젝트입니다.

## 🔍 주요 기능

- **운세 테스트** — 재물운·사랑운·성공운·우정운 랜덤 출력
- **심리 테스트 4개** — 동물·색깔·루틴·행운 아이템 (8문항·4답변 구조)
- **방명록** — Firebase Firestore 실시간 저장·조회, 공개/비밀 댓글 분리
- **응원글귀** — 사용자 입력 → Firebase 저장 → 검수 후 반영
- **타이핑 게임** — 난이도 2단계, 글자 수별 점수 가중치, 30초 제한

## 🛠️ 사용 기술

| 구분 | 내용 |
|------|------|
| 프론트엔드 | HTML · CSS · JavaScript |
| 데이터베이스 | Firebase Firestore |
| 배포 | Netlify |

## 📊 구현 흐름

1. **실시간 DB** — 사용자 입력 → Firebase Firestore 저장 → 화면 즉시 반영
2. **비실시간 DB** — 응원글귀 입력 → 검수 후 JS 직접 삽입 → 재배포
3. **운세 로직** — Math.random() 기반 퍼센트 및 메시지 랜덤 출력
4. **심리 테스트** — 답변별 다차원 점수 매핑 후 결과 도출
5. **보안 처리** — 우클릭·단축키 차단, Firebase 보안 규칙 적용

## 💡 배운 점

- 데이터 생성 → 저장 → 조회 → 화면 표시 전체 흐름 직접 설계
- 실시간(Firestore)과 비실시간 데이터 파이프라인 차이 이해
- API 키 보안 이슈 직면 후 대안 설계 경험
- 모바일 미대응 한계 확인 → 추후 개선 필요성 인식
